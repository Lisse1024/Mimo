# KOC LangGraph Agent 60-90 秒演示脚本

## 演示前准备

1. 启动应用：`npm run dev`
2. 打开桌宠主窗口，输入 `演示路径`，先让现场看到内置路演路线。
3. 复制 `demo/stable-single-work-material.json` 里的 `user_message`，发起单条作品拆解。
4. 如果现场网络或模型慢，直接点“查询结果”或展示已完成的最近任务；核心是展开 trace、证据与复盘闭环。

## 口播脚本

**0-15 秒：定位**

这是一个面向 KOC 创作者的桌面增长 Agent。它不是普通聊天页，而是会观察当前页面、接收截图/录屏/链接，把素材放进 LangGraph 工作流里做证据判断和策略规划。

**15-35 秒：发起任务**

我现在用一条固定作品素材演示：一个冷启动账号的早餐饭团短视频。Agent 会先识别这是单条作品拆解，不会直接上升成账号长期定位，然后把页面可见指标、视频结构、评论线索和用户目标汇总成证据快照。

**35-60 秒：展示 Agent 能力**

这里重点看“智能体执行轨迹”：平台线索解析、素材处理、证据快照、decision gate、策略生成、结果写入，每一步都有状态。它不是一次性 prompt，而是 LangGraph 根据证据质量路由；证据不足时会降级或要求补证据，不会编造后台数据。

**60-78 秒：展示结果**

结果会给出为什么这条视频可能有收藏和复刻价值，以及下一条能直接拍的脚本。再展开“证据与风险”，可以看到哪些来自用户输入、哪些是页面可见指标、哪些只是模型推断。

**78-90 秒：复盘闭环**

最后我输入发布后数据，比如播放、点赞、收藏、评论和涨粉。Agent 会把这次实验写入长期记忆，下一轮同类账号或同类作品分析会优先引用有效模式和失败教训。

## 备用复盘输入

```text
复盘：播放 12800，点赞 860，收藏 420，评论 53，分享 31，涨粉 46，效果不错，收藏主要来自早餐做法和可替换口味。
```

## 评委追问回答

如果被问“Agent 体现在哪里”，回答：

这里有三层。第一层是工具：窗口观察、浏览器线索、截图/录屏、OCR/ASR、记忆读写。第二层是图：LangGraph 的 decision gate 会根据任务类型、证据质量和记忆信号选择继续分析、补证据或暂停。第三层是闭环：建议不是一次性答案，而是会变成可复盘实验，发布后指标会写入长期记忆。
